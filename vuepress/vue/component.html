# 制作组件发布NPM

使用Vue-cli 3.x 自制通用化组件,并发布到NPM服务器

# 安装

安装和配置Vue-cli 不再赘述,去Vue-cli官网看就好了

# 结构

你可以仿照如下结构(只列必要的部分)来创造你的组件

|-your-component-folder
|--dist           # webpack 打包文件存放目录
|--node_modules   # webpack 自动创建
|--public         # vue-cli 3.x 版本自动生成放置公共资源目录
|--src            # 项目具体文件夹
|---assets        # vue-cli 3.x 版本自动生成 项目资源
|---packages      # 仿照Element-ui 目录结构,存放各个分组件 类似 /components
|---plugins       # 用于引入外部插件的文件
|---index.js      # 组件入口文件 原入口应为 main.js
|--package.json   # 项目打包的必要文件
|--vue.config.js  # vue-cli 3.0 的配置文件 配置请前往 vue-cli 官网查看

# 组件制作

根据上述文件结构可以清楚的知道组件应该放在 /src/packages 文件夹下

# 创建组件

在 /src/packages 目录下创建组件 (假设你的小组件名称为 hello-word),目录结构将如下

|-packages
|--hello-word         # 组件文件夹
|---index.js          # 组件export的暴露，用于其他被其他项目单个组件引入
|---hello-word.vue    # 具体的组件Vue业务文件

建议

当一个组件中包含多个小组件或者该组件有子组件，建议在当前组件文件夹下新建 src 目录,将组件具体实现文件放置在 src 目录下,组件目录只有一个 index.js 文件，用于暴露主组件。

当发现需要暴露多个子组件时，这时候要考虑拆分文件夹解耦了。

子组件暴露只需新建子组件名称的文件夹,进行暴露 最后结构有可能是这样的：

|-packages
|--parent           # 父组件暴露文件夹
|---src             # 组件实现目录
|----parent.vue
|----parent-child.vue
|---index.js
|--parent-child       # 子组件暴露文件夹
|---index.js

# 批量引入

整个项目的入口文件 /src/index.js 可以暴露整个组件内容，方便外部项目统一引入

它的文件可能是这样的

import HelloWord from "./packages/hello-word";

const components = [HelloWord];

const install = function(Vue) {
  components.forEach(component => {
    Vue.component(component.name, component);
  });
};

if (typeof window !== "undefined" && window.Vue) {
  install(window.Vue);
}

// export 中install 统一注册了组件，也可以单个暴露组件方便其他项目按需使用
export default {
  install,
  HelloWord
};

# 打包配置

  • Vue.config.js

vue-cli 3.0的统一配置文件

因为vue-cli 默认配置是以 /src/main.js 作为入口 我们需要将其修改成 /src/index.js

那我们的 vue.config.js 可能是这样的

// vue.config.js
module.exports = {
  publicPath: "/",
  // 将构建好的文件输出到哪里
  outputDir: "dist",
  devServer: {
    port: 8081
    // host: "192.168.6.123"
  },
  // webpack 配置项
  configureWebpack: {
    entry: { app: "./src/index.js" }
  }
};

# 发布

  1. npm run build 创建文件

  2. npm publish 进行发布

  3. npm unpublish your-component-name --force 删除已发布的组件

# 踩坑

  • 发布上去后,其他项目import 安装完成后 提示未找到文件

    因为 npm publish 默认不会将dist 目录添加到包中，解决方案：

    1. 将文件打包位置更换为其他目录,并修改 package.json 文件的main节点指向位置

      更换目录方式 可在 vue.config.js 文件中outputDir节点修改，也可自定义命令指定目录

    2. 在package.json 文件中 加入 files 节点，将 dist目录包含进去
